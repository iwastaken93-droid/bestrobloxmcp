# Security Policy

## Supported Versions

We release security patches for the latest minor version of BestRobloxMCP. Always update to the latest version to receive security fixes.

| Version | Supported          |
| ------- | ------------------ |
| 2.x     | :white_check_mark: |
| < 2.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in BestRobloxMCP, please report it responsibly:

1. **Do NOT** open a public issue — this could expose the vulnerability before a fix is ready.
2. Email the details to the repository owner or open a **private security advisory** on GitHub.
3. Include a clear description of the vulnerability, steps to reproduce, and the impact.
4. Allow up to **72 hours** for an initial response.
5. We will work with you to verify, patch, and disclose the fix in a coordinated manner.

## What We Promise

- Acknowledge receipt of your report within 72 hours.
- Investigate and provide a timeline for a fix.
- Credit you in the release notes (unless you prefer to remain anonymous).
- Never take legal action against good-faith security research.

## Scope

This policy covers:
- The `@bestrobloxmcp/bestrobloxmcp` npm package
- The `@bestrobloxmcp/core` internal package
- The Studio plugin (`MCPPlugin.rbxmx`)
- The web dashboard served by the MCP server

Out of scope:
- Third-party dependencies (report to their respective maintainers)
- Roblox Studio itself (report to Roblox)
